MCP：AI Agent 的 USB 接口

2026-03-24 · protocol · ecosystem · MCP · 10 min read

一句话解释 MCP

如果你曾经困惑"AI Agent 怎么才能安全、标准化地调用外部工具"，那 MCP 就是答案。

Model Context Protocol（MCP） 是 Anthropic 于 2024 年底提出的开放协议，它定义了一套标准接口，让 AI 模型（或 Agent）能够以统一方式连接各种数据源、工具和服务。用一个类比：MCP 之于 AI Agent，就像 USB 之于电脑——你不需要为每个外设写专属驱动，插上就能用。

为什么需要 MCP？

Agent 时代的碎片化困境

在 MCP 出现之前，每个 AI 应用接入外部工具的方式都是"各搞各的"：

  • ChatGPT 的插件系统：OpenAI 私有协议，只能在 ChatGPT 生态内用
  • LangChain 的 Tool：Python 绑定，换个框架就得重写
  • 各家自研 Function Calling：格式不同、参数不同、错误处理不同

这就像 90 年代的电脑接口——串口、并口、PS/2、SCSI，每个设备一个标准，用户头疼，开发者更头疼。

MCP 的出现就是为了解决这个碎片化地狱。它提供了一个协议层，让"AI 如何调用工具"这件事有一个通用答案。

从 N×M 到 N+M

没有 MCP 时，如果有 N 个 AI 应用和 M 个工具，理论上需要 N×M 个适配器。有了 MCP，只需要 N 个 MCP 客户端 + M 个 MCP 服务器 = N+M 个实现。复杂度从二次降到线性，这就是协议的价值。

MCP 架构详解

核心三要素

MCP 采用经典的 客户端-服务器（Client-Server） 架构：

┌─────────────┐     JSON-RPC 2.0     ┌─────────────┐
│  MCP Client  │ ◄──────────────────► │  MCP Server  │
│ （AI Agent）  │    stdio / SSE /     │ （工具提供方） │
│              │    HTTP Stream       │              │
└─────────────┘                      └─────────────┘
  1. MCP Host：运行 AI 模型的宿主环境（如 Claude Desktop、IDE 插件、自定义 Agent）
  2. MCP Client：Host 内部的协议客户端，负责与 Server 通信
  3. MCP Server：提供具体能力的服务端（文件系统、数据库、API 网关等）

三大能力支柱

1. Tools（工具）

Tool 就是一个可被 AI 调用的函数，有明确的输入输出 schema。AI 模型通过 function calling 机制决定何时调用哪个 Tool，MCP 负责将调用请求路由到正确的 Server。

2. Resources（资源）

Resources 允许 Server 暴露结构化数据供 AI 读取，比如文件内容、数据库 schema、API 文档等。与 Tools 不同，Resources 是被动的——AI 主动去"读"，而不是 Server 主动去"写"。

3. Prompts（提示模板）

Prompts 是预定义的提示模板，Server 可以提供高质量的 prompt 供 AI 使用。比如一个代码审查 Server 可以提供 code-review prompt，包含专家级的审查指令。

传输层：灵活的通信方式

传输方式适用场景特点
stdio本地工具最简单，进程间通信，低延迟
SSE远程服务HTTP 长连接，支持流式响应
HTTP Stream云服务基于 HTTP 的双向流，适合分布式部署

MCP 生态全景

官方 Server 参考实现

  • filesystem：安全的文件系统访问
  • git：Git 仓库操作
  • postgres / sqlite：数据库查询
  • fetch：HTTP 请求代理
  • memory：基于知识图谱的持久化记忆
  • puppeteer / playwright：浏览器自动化

客户端生态

  • Claude Desktop：Anthropic 官方客户端，原生支持 MCP
  • Cursor / VS Code：代码编辑器中的 AI 助手通过 MCP 连接开发工具
  • OpenClaw：开源 AI Agent 框架，MCP 作为核心工具集成方式
  • LangChain / LlamaIndex：主流 AI 框架添加了 MCP 适配器

安全：MCP 的隐忧与对策

  1. 权限过宽：遵循最小权限原则
  2. Prompt Injection：Host 环境应对 Server 内容做沙箱隔离
  3. 数据泄露：传输层加密（TLS）、访问控制
  4. 供应链攻击：使用签名验证、来源审查
MCP 安全最佳实践：永远不要在生产环境运行未经审查的第三方 MCP Server。像对待 sudo 权限一样对待 MCP 配置。

生态规模🔥 爆发增长📉 衰退📊 稳定🌱 早期

总结

MCP 不是又一个花哨的 AI 概念，而是一个解决真实问题的工程协议。它用标准化的方式，终结了 AI 工具集成的碎片化时代。就像 USB 统一了电脑外设接口一样，MCP 正在统一 AI Agent 的工具接口。
