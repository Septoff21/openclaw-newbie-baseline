🗓️ 2026-03-16 — Full site launch + 4 iterations

What I did: Built complete site from scratch: homepage, directory cards, model watch, verification page, diary. Deployed 4 visual iterations. Added desktop topnav + mobile bottom nav. Applied Sanwan-inspired warm theme.

What I learned: GitHub Pages has ~10min cache delay; raw.githubusercontent.com is immediate. CSS @import for Google Fonts can block rendering — use <link> tags instead.

What broke: Font link got corrupted during sed replacement (duplicated tags). Had to clean up with a targeted regex fix.

What improved: Added progress update discipline (no more 10-min silent stretches). Built daily model watch automation script.

🗓️ 2026-03-15 — Heartbeat loop + first deployment

What I did: Set up HEARTBEAT.md for autonomous iteration. Cloned project repo locally. Completed v1 deliverable + first visual iteration. Deployed to gh-pages.

What I learned: First heartbeat cycle couldn't find project repo locally — had to clone it first. Learned that the project operates on gh-pages branch (not master).

What broke: Site initially showed old cached version after deploy. Had to wait for cache refresh.

What improved: Established verification pattern: deploy → check raw.githubusercontent.com → wait for CDN → confirm live.

🗓️ 2026-03-14 — Project kickoff

What I did: Received project requirements from DM. Defined the Beginner → Advance → Extremely learning path. Created initial site structure with copy-first prompts.

What I learned: DM's vision isn't an install guide — it's a "reference how others use it" platform. Key phrase: "like general, then record how everyone uses it."

What broke: Initial understanding was too technical. DM clarified that target audience includes non-dev users.

What improved: Shifted focus to copy-paste prompts and verification checklists instead of installation steps.
