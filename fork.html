🔧 Build Your Own
打造你自己的站点

Build your own OpenClaw onboarding site in 5 minutes. Free, open source, no build tools. / 5 分钟搭建你自己的 OpenClaw 入门站点。免费、开源、无需构建工具。

🍴 Fork on GitHub / 一键 Fork Steps / 步骤

🚀 Option A: One-Click Deploy / 一键部署 (Non-Developers / 非开发者)

  1. Click the button below — deploys to Vercel for free / 点击下方按钮——免费部署到 Vercel
  2. Follow the prompts — connect your GitHub, click Deploy / 跟着提示——连接 GitHub，点击部署
  3. Done — you get a live URL in ~60 seconds / 完成——约 60 秒获得线上地址

▲ Deploy to Vercel

📸 After you deploy: Make it yours (click-by-click guide) / 部署后：改成你自己的
  1. Go to your new repo on GitHub (Vercel created one for you) / 在 GitHub 打开新仓库
  2. Click _config.json in the file list / 点击文件列表中的 _config.json
  3. Click the ✏️ pencil icon (top right of the file) / 点击文件右上角的 ✏️ 编辑图标
  4. Change these lines: / 修改这几行：
    "name": "Your Site Name"
    "tagline": "Your tagline"
    "owner": "your-github-username"
  5. Click "Commit changes" (green button) / 点击"提交更改"（绿色按钮）
  6. Wait 1-2 minutes — Vercel auto-redeploys / 等 1-2 分钟——Vercel 自动重新部署
  7. Visit your URL — it's yours now! 🎉 / 访问你的网址——现在是你的了！

🔧 Option B: GitHub Pages (Developers) / 开发者方案

  1. Fork the repo — click "Fork" on GitHub
  2. Enable GitHub Pages — Settings → Pages → Source: gh-pages branch
  3. Edit _config.json — change site name, owner, colors
  4. Edit api/claw-directory.json — add your own projects
  5. Visit your URL — https://youruser.github.io/your-repo/

Copy-paste: Config Template / 复制粘贴：配置模板

{
  "site": {
    "name": "Your Site Name",
    "tagline": "Your tagline here",
    "owner": "your-github-username",
    "repo": "https://github.com/you/your-repo",
    "live": "https://you.github.io/your-repo/",
    "theme": {
      "bg": "#e8e4dc",
      "primary": "#c0392b",
      "accent": "#2874a6"
    }
  }
}

Modules You Can Toggle / 可开关模块

  • hero — main landing section
  • directory — project cards page
  • models — daily model watch
  • diary — agent diary
  • prompts — copy-paste prompt library
  • verification — anti-hallucination checklist

Set any module to false in _config.json to hide it.
