# Sanwan.ai Case (English Adaptation)

Reference: https://sanwan.ai/

## What this project style does well
- Strong personal narrative + experiment framing
- AI-agent as the main operator (build/write/operate/promote)
- Public progress loop (daily logs + visible milestones)
- Clear social proof and momentum storytelling

## Core message (translated/adapted)
- “An AI assistant is a digital employee if you give it a computer and clear tasks.”
- The site is presented as an autonomous growth experiment.
- Daily operations include content production, SEO maintenance, and promotion loops.

## Productizable pattern for our project
1. **Narrative hook**: from injury/busy founder → AI delegation experiment
2. **Operating proof**: what AI did today (build, content, SEO, distribution)
3. **Evidence layer**: metrics, logs, and links
4. **Call to action**: copy prompt and reproduce the workflow

## Beginner-friendly English copy block
"OpenClaw can start as your beginner assistant, then evolve into an autonomous operator.
Copy one prompt, run one workflow, verify one result.
Repeat daily, and your system compounds."

## Prompt (English)
You are my OpenClaw operations assistant.
Goal: replicate a lightweight autonomous growth workflow.
Do now:
1) Run SEO baseline checks (meta/canonical/sitemap/robots)
2) Generate one publishable content draft from today’s changelog
3) Propose one low-risk distribution action
4) Return evidence checklist and next-day task plan
Format: plan/progress/blocker/final summary.

## Notes for adaptation
- Keep the tone warm and human, not overly technical.
- Show one concrete output each day.
- Prioritize reproducibility over hype.
