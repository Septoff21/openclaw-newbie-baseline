# Sanwan Diary Reverse-Engineering (English)

Reference: https://sanwan.ai/

## Why the diary format works
1. **Narrative hook**: personal constraint + bold experiment
2. **Daily evidence**: what AI changed today
3. **Transparent metrics**: traffic/progress indicators
4. **Audience participation CTA**: simple, low-friction action

## Reusable diary entry structure
- **Context**: what happened today
- **Action**: what the AI did (build/write/SEO/distribution)
- **Result**: measurable output (pages fixed, posts published, clicks)
- **Risk/Issue**: what went wrong or was noisy
- **Next step**: tomorrow’s single focus

## Reverse-engineered content loop
1. Collect today’s changelog
2. Ask AI to draft one diary post with evidence
3. Publish to one channel/site
4. Record feedback/signals
5. Feed learnings into tomorrow’s prompt

## Copy Prompt: diary mode
You are my OpenClaw growth diarist.
Use today's changelog to create one public progress post.
Output format:
1) Context (1-2 lines)
2) What AI changed today (bullet list)
3) Evidence and metrics (if available)
4) What failed/noise filtered
5) Tomorrow's single next action
Tone: warm, honest, non-technical, beginner-friendly.

## Beginner adaptation rule
- keep each post short
- show one real result
- avoid jargon
- end with one actionable CTA
