Verification Loop (anti-hallucination)

  • Always require: plan → progress → blocker → final summary.
  • Require command-level evidence for critical actions.
  • Context warning policy: 70% / 85% / 95%.
  • If unverifiable, ask for retry with explicit checks.

Verification Checklist (practical)

  1. Scope lock: restate goal in one sentence before execution.
  2. Evidence lock: include exact commands/URLs used.
  3. Result lock: show before/after or status code proof.
  4. Risk lock: mention blocker and rollback when relevant.

Model Context Reminder (before you trust output)

  • Different model, different limits: context window and tool behavior vary by model/provider.
  • Always ask for evidence: command output, URL proof, or before/after screenshots.
  • Use the warning ladder: 70% / 85% / 95% context risk notifications.

Tip: if results look uncertain, retry with tighter scope and explicit verification steps.
