Why Verify? / 为什么要验证？

AI agents can sound confident while being wrong. This checklist adds a verification layer so you can trust the output.
AI 可能自信满满地说错话。这份清单增加验证层，让你能信任输出。

  • Hallucination / 幻觉风险：models may invent URLs, commands, or facts / 模型可能编造 URL、命令或事实
  • Context / 上下文限制：different models behave differently at high usage / 不同模型在高使用率下表现不同
  • Tool failures / 工具失败：tool calls can silently fail or produce partial results / 工具调用可能静默失败

4-Lock Verification

🔒
Scope Lock

Restate the goal in one sentence

🔍
Evidence Lock

Exact commands, URLs, file paths

📊
Result Lock

Before/after or status code proof

⚠️
Risk Lock

Blockers and rollback plan

Copy-paste: Verification Prompt

After completing the task, provide:
1. Scope lock: What was the original goal?
2. Evidence lock: List all commands/URLs used with exact output
3. Result lock: What changed? Show before/after proof
4. Risk lock: Any blockers? What's the rollback plan?

If any lock is missing, say "INCOMPLETE" and explain what's missing.

Context Warning Ladder

When your context usage hits these thresholds, take action:

LevelThresholdAction
🟡 Caution70%Start summarizing, prepare to compress
🟠 Warning85%Compress context, drop non-essential history
🔴 Critical95%Stop, save state, start fresh session

🚩 Red Flags (immediately re-verify)

  • ❌ Agent claims a URL is live but won't show status code
  • ❌ Agent says "it works" without showing output
  • ❌ Agent skips the plan/progress/blocker/summary flow
  • ❌ Agent uses vague phrases like "should be fine" or "probably works"
  • ❌ Agent refuses to provide verification commands
